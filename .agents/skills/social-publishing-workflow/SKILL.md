# Social Publishing Workflow

本 skill 是社交内容从“原始想法”到“平台发布”的默认入口和上层编排器。它不替代具体写作、平台适配或平台发布 skill，而是让用户在实际使用时感受到一条清晰、主动、可校准的流程。

## 硬约束

- 只要用户意图是推进一篇社交内容，不论是“写一篇”“继续昨天那篇”“做预览”“发布草稿”“发到某个平台”，都优先进入本 skill，并先判断当前流程阶段。
- 每次响应都要明确当前阶段、已完成产物、当前阻塞、建议下一步；不要让用户自己回忆下一步该做什么。
- ArcSocial 项目根目录只存放开源项目能力和治理产物。用户内容、草稿、平台稿、素材、指标和个人发布产物必须读写数据工作区。
- 默认数据工作区是 `workspace/`；如果项目根目录存在 `arcsocial.config.json`，读取其中的 `workspacePath`。
- 本 skill 负责流程编排，不直接替代 `social-writing`、`wechat-draft` 或未来平台 adapter 的具体执行规则。
- 平台能力必须可扩展。不要把公众号当作唯一平台；公众号只是当前已有完整 adapter 的平台。
- 没有平台 adapter 时，不硬凑发布流程；完成通用写作和平台稿后，报告缺少 adapter，并把缺口作为后续能力记录。
- 远程发布、草稿箱写入、公开发布、凭证输入和破坏性操作必须遵守对应平台 skill 的确认门禁。
- 用户做过的人工校准必须进入下一阶段判断：内容方向、排版参考、封面裁切、列表安全、平台草稿问题和中间产物简化都不能只停留在对话解释里。

## 主流程

每次处理社交内容发布任务时，按下面阶段推进。可以跳过已经完成且有文件证据的阶段，但必须说明依据。

### 0. 建立流程面板

输入：用户请求、当前对话、仓库文件、已有平台稿或发布产物。

动作：

- 判断目标内容对象：新文章、已有草稿、平台稿、预览产物，还是发布结果。
- 判断目标平台：公众号、X、LinkedIn、小红书、知乎，或暂未指定。
- 查找已有产物：`<workspacePath>/inbox/ideas/`、`<workspacePath>/content/drafts/`、`<workspacePath>/platforms/<platform>/`、`<workspacePath>/publishing/<platform>/`。
- 向用户报告一个简短流程面板：
  - 当前阶段。
  - 已有产物。
  - 缺失产物。
  - 本轮建议推进到哪一步。

退出条件：明确当前对象、平台、阶段和本轮目标。如果平台或文章对象无法合理判断，问一个最小澄清问题。

### 1. 原始输入和标准草稿

输入：原始想法、粗稿、用户补充，或已有 `<workspacePath>/content/drafts/`。

动作：

- 使用 `social-writing` 记录原始输入、创建或更新标准草稿。
- 保留用户原始表达，不把平台稿当作唯一事实来源。
- 如果用户是在已有文章上继续校准，优先更新对应草稿或平台稿，不新建重复文章。
- 内容方向发生变化时，明确报告变化原因和影响范围。

退出条件：存在可追溯的原始记录和标准草稿，或明确说明为什么本轮只记录不成稿。

### 2. 内容校准门禁

输入：标准草稿、用户反馈、历史校准点。

动作：

- 主动复查核心观点、用户原始要点和本轮新增建议是否都进入草稿。
- 把用户的人工校准分类为：主题定位、核心观点、表达口吻、结构顺序、事实风险、平台偏好。
- 对缺失或冲突之处直接提出最小修正建议。
- 只有内容方向基本稳定后，才推进平台适配和发布准备。

退出条件：用户接受当前内容方向，或本轮完成了明确的内容修正并等待下一次校准。

### 3. 平台适配

输入：标准草稿、目标平台、平台偏好。

动作：

- 使用 `social-writing` 生成或更新 `<workspacePath>/platforms/<platform>/YYYY-MM-DD-slug.md`。
- 如果目标平台是公众号，后续发布准备交给 `wechat-draft`。
- 如果目标平台没有 adapter，完成平台稿后停止在“平台稿就绪”，报告缺少发布 adapter 的产品缺口。
- 不同平台的标题、摘要、段落节奏、图片需求和交互口径可以不同，但不能丢失标准草稿的核心观点。

退出条件：平台稿存在，并且清楚记录平台、路径和下一步发布准备方式。

### 4. 平台预览和素材门禁

输入：平台稿、平台 adapter、图片资产、历史发布偏好。

动作：

- 对有 adapter 的平台，调用对应发布 skill 生成本地预览或发布准备产物。
- 对公众号，使用 `wechat-draft` 自动完成发布层排版、安全列表转换、最近发布文章风格参考、封面质量检查和完整本地预览。
- 主动检查素材问题：封面是否适配平台裁切、是否含不可读小字、正文图片是否必要、图片路径是否可发布。
- 主动检查格式问题：列表、标题层级、摘要、字段长度、本地预览和平台草稿的差异风险。

退出条件：本地预览可供用户检查；如果素材或格式不满足发布门禁，停止并说明必须先修什么。

### 5. 人工预览校准

输入：本地预览、报告、用户反馈。

动作：

- 引导用户检查内容、封面、摘要、插图、排版和平台字段。
- 用户指出的问题要落到对应层：
  - 内容问题回到标准草稿或平台稿。
  - 排版和平台渲染问题回到平台发布 skill 或脚本。
  - 素材问题回到资产生成或选择。
  - 流程问题回到本 skill 或对应 skill 的契约。
- 修完后重新生成预览，不直接跳到发布。

退出条件：用户确认预览可以进入发布准备，或明确要求停止。

### 6. 发布准备和远程写入

输入：已确认的本地预览、平台发布产物、凭证状态、用户确认。

动作：

- 使用对应平台发布 skill 准备远程写入。
- 远程草稿、远程发布、凭证输入和 API 调用前，遵守平台 skill 的确认门禁。
- 成功后记录非敏感结果摘要到 `<workspacePath>/publishing/<platform>/`。
- 失败时报告缺少的前置条件、已生成的本地产物和下一步动作。

退出条件：远程草稿或发布结果已记录，或明确停在缺少凭证、缺少确认、缺少 adapter 等阻塞状态。

### 7. 发布后复盘沉淀

输入：发布结果、用户人工校准、过程问题。

动作：

- 复盘这次流程中用户被迫主动提醒的步骤。
- 判断问题属于内容经验、平台适配经验、发布脚本缺陷、素材生成偏好，还是上层流程编排缺口。
- 如果问题会在同类任务中重复出现，建议更新对应 skill 或脚本。
- 复盘时区分“本轮已修复”和“后续产品缺口”。

退出条件：给出本轮产物摘要、已沉淀规则、剩余缺口和下一次同类任务应自动执行的动作。

## 平台路由

- 通用记录、标准草稿、平台稿：使用 `social-writing`。
- 公众号预览、Wenyan 输入、草稿箱准备和发布层安全处理：使用 `wechat-draft`。
- ArcSocial 项目根目录的 `arckit/` 只存放开源项目治理产物，不存放个人文章发布产物。
- 其他平台：如果已有对应平台 skill，按该 skill 的门禁执行；如果没有，停在平台稿就绪，并记录 adapter 缺口。

## 最终汇报字段

每次使用本 skill 后，汇报：

- `content`: 当前文章或内容对象。
- `platform`: 目标平台，或未指定。
- `stage`: 当前阶段。
- `completed`: 本轮完成了哪些阶段。
- `artifacts`: 关键产物路径。
- `needs_user_review`: yes 或 no。
- `blocked_by`: 阻塞原因；没有则写 none。
- `next_step`: 下一步建议动作。
- `workflow_gap`: 本轮发现的流程缺口；没有则写 none。
