# Content Workflow

This is an ArcSocial project-level playbook. User content is read from and
written to `<workspacePath>`, which defaults to `workspace/` and can be changed
in `arcsocial.config.json`.

## 1. Capture

Put raw thoughts, references, screenshots, and materials into
`<workspacePath>/inbox/`.

## 2. Select

Promote useful inputs into `<workspacePath>/content/drafts/` when they have a
clear audience or angle.

## 3. Develop

Use the draft template to clarify the core idea, supporting material, and canonical copy.

## 4. Adapt

Create platform-specific versions under `<workspacePath>/platforms/` or
`<workspacePath>/content/ready/`.

## 5. Publish

Use the project-level checklist
`playbooks/checklists/publish-checklist.md` before posting.

## 6. Learn

Record metrics, comments, and follow-up ideas in
`<workspacePath>/data/metrics/` and `<workspacePath>/playbooks/experiments/`.
